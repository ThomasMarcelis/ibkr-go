# Feature Matrix

Comparison of existing Go libraries for the Interactive Brokers TWS and IB Gateway socket API against the `ibkr-go` v1 target. This document exists as anti-pattern study, not as an endorsement. Assessment is based on the public README and repository structure of each library as observable in April 2026. Cells marked `unverified` could not be confirmed from public documentation alone and should not be taken as claims about what the library does or does not do internally.

| Library | Public API shape | Session semantics | Reconnect handling | Testing substrate | Idiomatic Go | Activity |
|---|---|---|---|---|---|---|
| [hadrianl/ibapi](https://github.com/hadrianl/ibapi) | `IbWrapper` callback interface the user implements, `NewIbClient(&Wrapper{})`, explicit `Connect`/`HandShake`/`Run`/`Disconnect` | unverified | unverified | unverified | low — forces callback inheritance as the primary user model | unmaintained (last push 2023-09-27) |
| [scmhub/ibapi](https://github.com/scmhub/ibapi) | `NewEClient(nil)`, EClient-shape mirror; self-described as "designed to mirror the official Python or C++ tws-api"; partially adapted from hadrianl/ibapi per its README | unverified | unverified | unverified | low — explicit goal is to mirror the official shape, which carries the same design gravity as the official client | active (last push 2026-04-01) |
| [scmhub/ibsync](https://github.com/scmhub/ibsync) | Synchronous wrapper built on `scmhub/ibapi`, inspired by `ib_insync`; `NewIB()`, functional options `NewConfig(WithHost, WithPort, WithClientID, WithTimeout)`; synchronous result methods (`AccountValues`, `AccountSummary`, `Portfolio`, `Positions`); channel-based streams (`PositionChan`, `ReqHistoricalData` returns a channel) | unverified | unverified | unverified | medium — functional options and typed synchronous methods are idiomatic, but the correctness floor is set by the `scmhub/ibapi` layer underneath | active, beta |
| [portfoliome/ib](https://github.com/portfoliome/ib) | Pure-Go; offers both low-level types and high-level `Manager` types | unverified | unverified | Ships a `testserver/` directory, but per the README the tests still require a live IB Gateway | unverified | effectively abandoned (last push 2014-03-28, LGPLv3 with static-linking exception) |
| **`ibkr-go` (v1 target)** | Typed one-shot request methods and typed subscriptions; explicit session info and subscription lifecycle (`Events`, `Done`, `Wait`, `Close`); no callback inheritance as the primary public shape | First-class observable session state, explicit negotiated version, explicit handshake readiness | Explicit per-subscription resume policy; no silent reconnect | In-process `testhost` package plus checked-in deterministic transcripts; live Gateway is only for transcript refresh | high — concrete types, functional options, errors-by-type | active (initial release in progress) |

## What the clean-room boundary permits

This matrix is drawn from public READMEs and repository metadata as anti-pattern study, in line with the provenance policy in `docs/provenance.md`. No code was copied from any of these libraries. Contributors referencing this matrix to inform design decisions should read the other libraries' READMEs and package layouts, not their source.
