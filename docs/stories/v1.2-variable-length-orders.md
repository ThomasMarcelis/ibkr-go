# v1.2: grounded advanced-order support

## Summary

v1.2 lands the grounded advanced-order subset that is justified by the
current codebase, live captures, and the shipped public API:

- `OpenOrder` keeps the existing fixed-layout fast path for simple
  169-field orders.
- Non-simple `OpenOrder` messages now decode grounded combo, algo, and
  order-condition sections instead of returning an unconditional partial
  parse.
- `PlaceOrder` now encodes BAG combo legs, order-combo prices, smart
  combo routing params, algo params, and grounded order conditions.
- Historical bid/ask and trade ticks now expose their tick attribute
  bitmasks end to end.

This is intentionally not the full rare-order wire surface. Delta-neutral
extensions, scale extensions, and the remaining ungrounded order-detail
branches are deferred to v1.3.

## Grounded Boundary

### Landed in v1.2

- BAG combo leg placement and observation.
- Order-combo leg prices and smart combo routing params.
- Algo strategy plus algo params.
- Grounded order conditions:
  - Price
  - Time
  - Margin
  - Execution
  - Volume
  - PercentChange
- `ConditionsIgnoreRTH` and `ConditionsCancelOrder`.
- `Commission`, `MinCommission`, `MaxCommission`, and
  `CommissionCurrency` on observed open orders.
- `TickAttrib` on historical bid/ask and trade ticks.

### Deliberately deferred to v1.3

- Delta-neutral order extensions.
- Scale order extensions.
- PEG BENCH / PEG BEST / PEG MID specialized fields.
- Full `CompletedOrder` detail extraction.
- Remaining ungrounded `OpenOrder` branches beyond the grounded subset.

## Implementation Notes

### OpenOrder

- The simple 169-field layout is still decoded via the existing fixed
  skip-based path because it is capture-grounded and stable.
- Expanded orders use a sequential decoder for the grounded variable
  sections.
- If an `OpenOrder` enters a deferred branch such as delta-neutral or
  scale extensions, the decoder still returns the safe partial result
  rather than guessing shifted downstream offsets.

### PlaceOrder

- BAG combo sections are encoded only when needed.
- Algo params are encoded only when `AlgoStrategy` is set.
- Conditions are encoded from structured public types; counts are
  derived from slices rather than caller-supplied count strings.

### Historical ticks

- `tickAttribBidAsk` and `tickAttribLast` are now decoded and encoded
  instead of being hardcoded to zero or skipped.
